## 测试 1: 验证自动备份调度器启动

**验证点**:
- ✅ 看到 "启动数据库备份调度器" 日志
- ✅ 显示下次备份时间和延迟分钟数
- ✅ 延迟时间计算正确（距离下一个15:30的时间）

## 测试 2: 手动触发备份（快速验证）

**目标**: 测试手动备份功能是否正常工作

**步骤**:
1. 打开设置页面
2. 找到"备份数据库"按钮
3. 点击按钮
4. 观察UI反馈和日志

**预期结果**:
- **UI**: 
  - 按钮变为"备份中..."并禁用
  - 显示Toast通知："备份成功"
  - 按钮恢复可用状态
- **日志**:
  ```
  ✅ 备份成功: C:\Users\XXX\.stockanalyzer\stockanalyzer_20260429.db
     文件大小: 2.35 MB
     耗时: 125ms
  ```
- **文件系统**:
  - 在 `~/.stockanalyzer/` 目录中看到新的备份文件

**验证点**:
- ✅ 备份文件创建成功
- ✅ 文件名格式正确（`stockanalyzer_YYYYMMDD.db`）
- ✅ 文件大小合理（与主数据库文件接近）
- ✅ 备份耗时 < 5秒

**目标**: 验证同日多次备份的序号追加机制

**验证点**:

- **后续点击**: 按钮保持禁用，无响应
- **日志**:
  ```

  ```

**验证点**:
- ✅ 同时只有一个备份在执行
- ✅ 后续点击被正确忽略
- ✅ 日志记录警告信息

## 测试 6: 过期备份清理

**目标**: 验证超过3天的备份被自动删除

**步骤**:
1. 手动创建几个旧的备份文件（修改系统时间或使用touch命令）
   ```bash
   # PowerShell示例
   $oldFile = "stockanalyzer_20260425.db"
   New-Item $oldFile -ItemType File
   # 修改文件时间为4天前
   (Get-Item $oldFile).LastWriteTime = (Get-Date).AddDays(-4)
   ```
2. 执行一次新备份
3. 检查旧文件是否被删除

**预期结果**:
```
🗑️ 已删除过期备份: stockanalyzer_20260425.db
清理完成: 删除了 1 个过期备份文件
```

**验证点**:
- ✅ 超过3天的备份文件被删除
- ✅ 3天内的备份文件保留
- ✅ 日志记录删除的文件数量

## 常见问题排查

### 问题1: 看不到备份调度器启动日志

**可能原因**:
- 日志级别设置过高
- backupService未正确导入

**解决方法**:
```typescript
// 检查 electron/index.ts
import { startBackupScheduler } from './services/backupService';
// 确保在 app.whenReady() 中调用
startBackupScheduler();
```

### 问题2: 备份文件找不到

**检查位置**:
```bash
# Windows
C:\Users\<用户名>\.stockanalyzer\

# 查看文件
ls ~/.stockanalyzer/*.db
```

### 问题3: 手动备份按钮无响应

**检查清单**:
1. IPC handler是否正确注册（`electron/index.ts`）
2. 渲染进程是否正确调用（`src/composables/useBackup.ts`）
3. 浏览器控制台是否有错误
