# Feature Specification: 持仓页面

**Feature Branch**: `007-position-page`
**Created**: 2026-04-04
**Status**: Draft
**Input**: User description: "实现持仓页面，初始情况下，只展示上半部分的持仓列表，当点击某只持仓股票时，显示该股票下方的交易记录。持仓列表和交易记录都查询自trade_record表。页面中支持新增交易记录，输入交易的股票代码、股票名称、交易类型（买入、卖出、股息）、交易日期、交易价格、交易数量。系统自动计算持仓均价和持仓数量后，保存到表中。股票价格计算方法参照文件"

## User Scenarios & Testing

### User Story 1 - 查看持仓列表 (Priority: P1)

作为用户，我需要在一个页面上查看我的持仓列表，以便快速了解我当前持有的股票概况。

**Why this priority**: 持仓列表是用户进入持仓页面的首要目的，需要首先展示

**Independent Test**: 可以通过打开持仓页面，验证列表是否正确显示所有持仓股票

**Acceptance Scenarios**:

1. **Given** 用户已打开持仓页面，**When** 页面加载完成，**Then** 应显示所有有持仓的股票（holding_count > 0），每只股票显示股票代码、股票名称、最近交易日期、持仓数量、持仓成本（持仓均价）、实时股价、盈亏金额、盈亏比例
2. **Given** 用户已打开持仓页面，**When** 页面加载完成，**Then** 系统应一次性从远程API获取所有持仓股票的实时股价，并计算盈亏金额和盈亏比例
3. **Given** 持仓列表已显示，**When** 用户切换到其他页面再返回持仓页面，**Then** 系统应重新从远程API获取最新股价并更新盈亏信息
4. **Given** 用户已打开持仓页面，**When** 用户点击某只持仓股票，**Then** 应展开显示该股票下方的交易记录列表
5. **Given** 用户已打开持仓页面，**When** 用户再次点击已展开的股票，**Then** 应收起交易记录列表
6. **Given** 用户点击某只持仓股票，**When** 系统查询该股票的交易记录时，**Then** 应先找到该股票上次holding_count为0的交易记录，如果不存在则查询所有交易记录；如果存在，则只查询交易时间晚于上次holding_count为0的交易记录的交易时间的交易记录
7. **Given** 系统显示交易记录列表，**When** 列表展示时，**Then** 交易记录应按交易时间倒序排列，且交易时间只展示到天（忽略时分秒）

---

### User Story 2 - 查看交易记录 (Priority: P1)

作为用户，我需要查看某只股票的完整交易记录，以便了解该股票的买入卖出历史。

**Why this priority**: 交易记录是理解持仓变化的关键数据

**Independent Test**: 可以通过点击持仓股票，验证交易记录是否正确显示

**Acceptance Scenarios**:

1. **Given** 用户已打开持仓页面，**When** 用户点击某只持仓股票，**Then** 应显示该股票的所有交易记录，按交易日期倒序排列
2. **Given** 用户已展开某只股票的交易记录，**When** 用户点击另一只股票，**Then** 原股票应收起，新股票的交易记录应展开

---

### User Story 2.1 - 编辑交易记录 (Priority: P1)

作为用户，我需要编辑已有的交易记录，以便修正输入错误的交易信息。

**Why this priority**: 用户可能会输入错误，需要能够修改

**Independent Test**: 可以通过点击编辑按钮，修改交易信息并保存

**Acceptance Scenarios**:

1. **Given** 用户已展开某只股票的交易记录，**When** 用户点击某条交易记录的编辑按钮，**Then** 应显示编辑表单，并带入该记录的原始值（股票代码、股票名称、交易类型、交易日期、交易价格、交易数量、持仓数量、持仓均价）
2. **Given** 用户已打开编辑表单，**When** 用户修改交易信息并保存，**Then** 应直接更新数据库中该记录，不进行持仓计算

---

### User Story 2.2 - 删除交易记录 (Priority: P1)

作为用户，我需要删除错误的交易记录，以便维护准确的交易历史。

**Why this priority**: 用户可能会输入错误的交易记录，需要能够删除

**Independent Test**: 可以通过点击删除按钮，删除交易记录

**Acceptance Scenarios**:

1. **Given** 用户已展开某只股票的交易记录，**When** 用户点击某条交易记录的删除按钮，**Then** 应从数据库中删除该记录

---

### User Story 3 - 新增交易记录 (Priority: P1)

作为用户，我需要新增交易记录，以便记录股票的买入、卖出或股息分红操作。

**Why this priority**: 这是记录交易的核心功能，没有此功能用户无法维护交易数据

**Independent Test**: 可以通过点击新增按钮，填写交易信息，验证交易记录被正确保存

**Acceptance Scenarios**:

1. **Given** 用户已打开持仓页面，**When** 用户点击新增交易记录按钮，**Then** 应显示交易记录编辑表单
2. **Given** 用户已打开交易记录编辑表单，**When** 用户填写股票代码、股票名称、交易类型（买入/卖出/股息）、交易日期、交易价格、交易数量并保存，**Then** 系统应自动计算持仓均价和持仓数量，保存到trade_record表
3. **Given** 用户已提交交易记录，**Then** 持仓列表应相应更新
4. **Given** 用户已打开交易记录编辑表单，**When** 用户输入股票代码并完成输入后（输入框失焦或按回车键），**Then** 系统应调用API获取该股票名称，并自动填入股票名称输入框

---

### User Story 4 - 自动计算持仓均价和数量 (Priority: P1)

作为用户，我希望系统自动计算持仓均价和持仓数量，以便准确反映我的实际持仓成本。

**Why this priority**: 这是自动化计算的核心功能，确保持仓数据准确

**Independent Test**: 可以通过新增交易记录，验证持仓均价和数量被正确计算

**Acceptance Scenarios**:

1. **Given** 用户新增一笔买入交易，**When** 系统保存交易记录后，**Then** 应自动计算并更新该股票的持仓均价和持仓数量
2. **Given** 用户新增一笔卖出交易，**When** 系统保存交易记录后，**Then** 应自动计算并更新该股票的持仓均价和持仓数量
3. **Given** 用户新增一笔股息分红，**When** 系统保存交易记录后，**Then** 应自动调整该股票的持仓均价（降低），持仓数量不变

---

## Requirements

### Functional Requirements

- **FR-001**: 系统 MUST 显示持仓列表，从trade_record表查询当前有持仓的股票（holding_count > 0）
- **FR-002**: 系统 MUST 在用户点击持仓股票时展开/收起该股票的交易记录
- **FR-003**: 系统 MUST 只同时展开一只股票的交易记录，点击另一只时自动收起前一只
- **FR-004**: 系统 MUST 显示交易记录列表，每条记录包含股票代码、股票名称、交易类型、交易日期、交易价格、交易数量、持仓数量、持仓均价
- **FR-005**: 用户 MUST 能够新增交易记录，输入股票代码、股票名称、交易类型（买入、卖出、股息）、交易日期、交易价格、交易数量。卖出交易时，交易数量输入负数
- **FR-005.1**: 系统 MUST 在用户输入股票代码并完成输入后（输入框失焦或按回车键），自动调用API获取该股票名称并填入股票名称输入框
- **FR-006**: 系统 MUST 在保存交易记录时自动计算持仓均价和持仓数量
- **FR-007**: 系统 MUST 按照股票价格计算方法计算持仓均价
- **FR-008**: 系统 MUST 将持仓均价和持仓数量保存到trade_record表的holding_price和holding_count字段
- **FR-009**: 系统 MUST 在进入持仓页面时，一次性从远程API获取所有持仓股票的实时股价
- **FR-010**: 系统 MUST 根据实时股价和持仓成本实时计算盈亏金额和盈亏比例
- **FR-011**: 用户 MUST 能够编辑已有的交易记录，编辑时带入原始值（包含持仓数量和持仓均价）
- **FR-012**: 系统 MUST 在编辑保存时直接更新数据库记录，不进行持仓计算
- **FR-013**: 用户 MUST 能够删除交易记录
- **FR-014**: 系统 MUST 在新增、编辑或删除交易记录后，自动刷新当前展开股票的交易记录列表，无需用户重新点击

### Key Entities

- **持仓股票**: 代表用户当前持有的股票，包含属性：股票代码、股票名称、最近交易日期、持仓数量、持仓成本（持仓均价）、实时股价、盈亏金额、盈亏比例
- **交易记录**: 代表用户的单笔交易，包含属性：股票代码、股票名称、交易类型（买入/卖出/股息）、交易日期、交易价格、交易数量、持仓数量、持仓均价

### Display Conventions

- **盈亏颜色**: 盈亏金额和盈亏比例为正数（盈利）时显示红色，为负数（亏损）时显示绿色

## Success Criteria

### Measurable Outcomes

- **SC-001**: 用户可在3秒内打开持仓页面并看到持仓列表
- **SC-002**: 点击持仓股票后，交易记录在1秒内显示
- **SC-003**: 新增交易记录后，持仓均价和持仓数量计算正确

## Assumptions

- trade_record表已存在且结构正确
- 股票价格计算方法参照用户提供的文件
- 系统支持买入、卖出、股息三种交易类型
- 持仓数量为0时，该股票不显示在持仓列表中
