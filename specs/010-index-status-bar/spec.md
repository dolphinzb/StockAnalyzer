# Feature Specification: 指数状态栏显示

**Feature Branch**: `010-index-status-bar`
**Created**: 2026-04-21
**Status**: Draft
**Input**: User description: "在每分钟获取监控股票数值的接口调用中，增加对上证指数、深成指数数据的获取；在窗口下方增加状态栏，用于显示上证指数、深成指数的数据，包括当前指数值、涨跌值、涨跌幅，上涨使用红色字体，下跌使用绿色字体。在涨跌幅数值前面，用向上或向下的箭头来表示上涨或下跌，箭头颜色与数值颜色一致。"

## Clarifications

### Session 2026-04-21

- Q: 指数持平（涨跌值为零）时，应该显示什么颜色和箭头？ → A: 持平时使用黑色字体，不显示箭头
- Q: 指数数据获取失败时，状态栏应显示什么内容？ → A: 显示上次成功获取的数据，并显示"数据更新失败"提示
- Q: 当涨跌幅数值非常大或非常小时，状态栏的布局如何处理？ → A: 缩小字体以适应宽度
- Q: 指数数据如何获取？ → A: 必须使用监控定时任务中调用API的接口，禁止单独调用接口
- Q: 手动刷新时是否也要更新状态栏指数数据？ → A: 当在自选股页面上点击手动刷新时，也要一并刷新状态栏指数数据
- Q: 应用启动后首次数据获取完成前，状态栏应显示什么内容？ → A: 显示占位符 "--"
- Q: 上证指数和深成指数在状态栏中如何排列？ → A: 左右分布，左侧上证指数，右侧深成指数
- Q: 状态栏的高度应该如何设计？ → A: 固定高度30px

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 查看市场整体走势 (Priority: P1)

用户在使用股票监控功能时，能够同时在窗口底部看到上证指数和深成指数的实时数据，从而快速判断市场整体走势。

**Why this priority**: 市场整体走势是投资者判断个股表现的重要参考背景，同时显示两大指数可帮助用户把握大盘情绪。

**Independent Test**: 用户打开应用后，无需任何操作，即可在窗口底部状态栏看到上证指数和深成指数的当前数值、涨跌值和涨跌幅。

**Acceptance Scenarios**:

1. **Given** 用户已启动应用，**When** 系统完成首次数据获取，**Then** 窗口底部状态栏显示上证指数和深成指数数据
2. **Given** 市场处于上涨状态，**When** 数据刷新后，**Then** 指数数值、涨跌值、涨跌幅均以红色显示，并在涨跌幅前显示向上箭头
3. **Given** 市场处于下跌状态，**When** 数据刷新后，**Then** 指数数值、涨跌值、涨跌幅均以绿色显示，并在涨跌幅前显示向下箭头
4. **Given** 市场处于持平状态，**When** 数据刷新后，**Then** 指数数值、涨跌值、涨跌幅均以黑色显示，不显示箭头

---

### User Story 2 - 实时跟踪指数变化 (Priority: P2)

指数数据随每分钟的数据刷新而自动更新，用户无需手动操作即可持续跟踪指数变化。

**Why this priority**: 实时跟踪指数变化是股票监控应用的核心价值之一，用户需要及时了解市场动态。

**Independent Test**: 用户观察状态栏，指数数据随每分钟的数据刷新而更新。

**Acceptance Scenarios**:

1. **Given** 状态栏已显示指数数据，**When** 系统执行下一分钟的数据获取，**Then** 状态栏中的指数数据自动更新为最新值
2. **Given** 指数数据正在显示，**When** 指数涨跌方向发生变化，**Then** 颜色和箭头随之变化
3. **Given** 状态栏已显示指数数据，**When** 用户在自选股页面点击手动刷新，**Then** 状态栏中的指数数据随股票数据一并刷新为最新值

---

### User Story 3 - 处理数据获取异常 (Priority: P3)

当指数数据获取失败时，系统应优雅地处理错误，向用户展示上次成功获取的数据并提示数据更新失败。

**Why this priority**: 保证用户持续看到有用的信息，同时了解数据可能不实时。

**Independent Test**: 在网络异常情况下，状态栏仍显示上次数据并有失败提示。

**Acceptance Scenarios**:

1. **Given** 状态栏正在显示指数数据，**When** 数据获取失败，**Then** 状态栏保持显示上次成功获取的数据，并在明显位置显示"数据更新失败"提示
2. **Given** 数据获取失败后，**When** 下次数据获取成功，**Then** 状态栏更新为最新数据，"数据更新失败"提示消失

---

### Edge Cases

- ~~当指数数据获取失败时，状态栏应显示什么内容？~~ → 显示上次成功获取的数据，并显示"数据更新失败"提示
- ~~当涨跌值为零时（指数持平），应该显示什么颜色和箭头？~~ → 使用黑色字体，不显示箭头
- ~~当涨跌幅数值非常大或非常小时，状态栏的布局如何处理？~~ → 缩小字体以适应宽度

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 系统必须在每分钟获取监控股票数据的同时，一并获取上证指数和深成指数的数据
- **FR-002**: 系统必须在窗口底部显示状态栏，用于展示指数数据
- **FR-003**: 状态栏必须显示每个指数的当前指数值、涨跌值、涨跌幅
- **FR-004**: 当指数上涨时，相关数值必须以红色字体显示，并在涨跌幅前显示向上箭头
- **FR-005**: 当指数下跌时，相关数值必须以绿色字体显示，并在涨跌幅前显示向下箭头
- **FR-006**: 当指数持平（涨跌值为零）时，相关数值必须以黑色字体显示，不显示箭头
- **FR-007**: 涨跌幅数值前必须显示向上箭头（上涨）或向下箭头（下跌），箭头颜色与数值颜色一致
- **FR-008**: 状态栏中的数据必须随每分钟的数据刷新自动更新
- **FR-008-1**: 当用户在自选股页面点击手动刷新时，状态栏指数数据必须一并刷新
- **FR-009**: 当指数数据获取失败时，状态栏必须显示上次成功获取的数据，并显示"数据更新失败"提示
- **FR-010**: 当涨跌幅数值超出状态栏可显示范围时，必须缩小字体以适应宽度
- **FR-011**: 应用启动后首次数据获取完成前，状态栏必须显示占位符 "--"
- **FR-012**: 状态栏必须采用左右分布布局，左侧显示上证指数，右侧显示深成指数
- **FR-013**: 状态栏必须使用固定高度30px

### Key Entities *(include if feature involves data)*

- **指数数据**: 包含指数名称（上证指数/深成指数）、当前指数值、涨跌值、涨跌幅、涨跌方向
- **状态栏**: 窗口底部的固定区域，用于实时展示指数信息
- **数据状态**: 包含正常、数据更新失败等状态标识

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 用户打开应用后5秒内，状态栏能够显示指数数据
- **SC-002**: 状态栏数据随每分钟的数据刷新同步更新，手动刷新时也一并更新
- **SC-003**: 指数上涨时，数值和箭头正确显示为红色；指数下跌时，正确显示为绿色；指数持平时，正确显示为黑色且无箭头
- **SC-004**: 涨跌方向变化时，颜色和箭头能够正确切换
- **SC-005**: 数据获取失败时，状态栏仍显示上次数据并有"数据更新失败"提示
- **SC-006**: 涨跌幅数值过大时，状态栏通过缩小字体确保不溢出

## Assumptions

- ~~假设指数数据获取接口与股票数据获取接口相同或可并行调用~~ → 必须使用监控定时任务中调用API的接口，禁止单独调用接口
- 假设涨跌方向以涨跌幅的正负来判断：正数为上涨，负数为下跌，零为持平
- 假设状态栏位于窗口底部固定位置，不影响主要监控区域的显示
- 假设涨跌幅显示格式为带正负号和百分比的数值（如 "+1.25%" 或 "-0.83%"）
- 假设"数据更新失败"提示应以醒目的方式显示，但不遮挡指数数据
