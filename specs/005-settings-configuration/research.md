# Research: 设置页面配置功能

**Branch**: `feature/005-settings-configuration`
**Date**: 2026-04-04

## 研究目标

评估实现设置页面配置功能的最佳技术方案。

## 技术选型

### 配置文件格式

| 方案 | 优点 | 缺点 | 决策 |
|------|------|------|------|
| JSON | 流行、TypeScript 原生支持 | 语法严格，用户不易手动编辑 | ❌ |
| INI | 简单、用户友好、广泛支持 | 无标准规范 | ✅ 采用 |
| YAML | 人类可读、表达能力强 | 依赖额外库 | ❌ |
| XML | 广泛使用 | 冗余、复杂 | ❌ |

**决策**: 使用 INI 格式，用户可以手动编辑配置文件。

### 配置文件存储位置

| 方案 | 优点 | 缺点 | 决策 |
|------|------|------|------|
| 可执行文件同目录 | 用户容易找到 | Windows 受 UAC 限制 | ❌ |
| 用户目录 ~/.stockanalyzer | 始终可写、跨平台一致 | 需要创建隐藏目录 | ✅ 采用 |
| AppData/Roaming | Windows 标准 | 路径较长 | ⚠️ 备选 |

**决策**: 使用 `~/.stockanalyzer/config.ini`，跨平台一致且始终可写。

### 配置加载时机

| 时机 | 优点 | 缺点 | 决策 |
|------|------|------|------|
| 应用启动时 | 配置始终最新 | 启动时间略有增加 | ✅ 采用 |
| 首次访问时 | 启动更快 | 需要懒加载逻辑 | ❌ |

**决策**: 应用启动时加载配置。

### 前端状态管理

| 方案 | 适用场景 | 决策 |
|------|----------|------|
| Pinia Store | 复杂状态、多组件共享 | ⚠️ 如需 |
| Vue Composition API | 简单状态、组件私有 | ✅ 采用 |
| Provide/Inject | 跨组件层级传递 | ⚠️ 如需 |

**决策**: 使用 Vue Composition API (useConfig)，简单高效。

## 安全考虑

### Electron 沙盒安全

- `contextIsolation: true` - 启用上下文隔离
- `nodeIntegration: false` - 禁用 Node 集成
- `sandbox: true` - 启用沙盒
- 通过 `contextBridge` 暴露有限的 API

### 文件系统访问

- 主进程拥有完整的文件系统访问权限
- 渲染进程通过 IPC 与主进程通信
- 配置文件位于用户目录，无需特殊权限

## 性能考虑

- 配置文件加载为同步 IO，单次操作 < 10ms
- 无需缓存，配置变更频率低
- UI 渲染使用 Vue 3 响应式系统，高效更新
